# F25 Requirements

## Objective
- The signup-to-programme lane must feel like one natural founder-demo workflow instead of a set of adjacent steps.

## Functional Requirements
- Billing must present one clear founding plan option for now, with no misleading multi-tier choice.
- Billing completion must route the merchant into first-programme setup explicitly.
- The workspace must support a compact guided setup flow for the first programme with these stages:
  - template
  - configure
  - publish and test
- The configure step must clearly indicate that saving completes setup and opens the daily-use programme operate surface.
- The operate step must expose the critical demo actions directly:
  - open customer join
  - copy the join link
  - stamp a visit
- Guided setup must be dismissible so the workspace can return to normal day-to-day usage.

## UX Requirements
- The guided setup must stay compact and horizontal, not revert into a large checklist.
- The billing screen must read as a real account step, not a generic mock-plan chooser.
- The first-programme flow must keep `programme` as the main object and treat the Wallet loyalty card as the current customer delivery output.
