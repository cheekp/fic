# F22 Acceptance

- The programme creation flow no longer reads as “create a loyalty card”; it reads as “create a programme from a template”.
- A merchant can see the difference between:
  - programme type
  - starter template
  - customer delivery type
  - current customer output
- The selected programme makes the current customer delivery output explicit without implying that output is the programme itself.
- The selected programme screen keeps Wallet as the live output without implying Wallet is the definition of the programme.
- The coffee-plus-food offer template remains available and reads as a distinct programme shape rather than a relabelled stamp card.
- Existing onboarding, auth, Wallet issuance, and Wallet update tests continue to pass.
- Automated tests cover the explicit template/type/delivery presentation for programme creation and configuration.
