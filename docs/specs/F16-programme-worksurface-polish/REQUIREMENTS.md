# F16 Programme Worksurface Polish Requirements

## Objective

Make the selected-programme workspace calmer and easier to use by reducing repeated framing, compressing secondary copy, and tightening the layout around the actual merchant tasks.

## Functional Requirements

### Operate

- the operate surface should keep customer join and stamping as the primary daily-use actions
- operate should remove unnecessary explanatory copy where the UI is already clear
- issued customer passes should remain available without feeling like a separate dashboard
- the selected-programme context should stay visible while keeping visual noise low

### Configure

- configure should present programme rule and wallet output as one coherent editing surface
- configure should reduce repeated headings, chips, and explanatory text
- the wallet loyalty card preview should remain visible, but should feel supportive rather than dominant

### General Programme Surface

- the selected-programme work area should feel like one focused surface, not stacked summaries
- non-essential chips and repeated labels should be reduced where they do not add decision value
- programme hierarchy from `F15` must remain intact

## Non-Goals

- changing the shop/programme hierarchy again
- adding new delivery types
- redesigning the public home or signup flow
