# F19 Acceptance

- A signed pass generated by the app contains:
  - `authenticationToken`
  - `webServiceURL`
  - the existing merchant and programme fields already used for the Wallet card
- A device can register for a pass serial through the Apple Wallet web-service route.
- The updated serial endpoint returns the changed pass serial after a visit is awarded.
- The updated pass endpoint returns a fresh `.pkpass` for that serial when authenticated.
- Visit-award and reward-redemption flows continue to work in the merchant workspace.
- Preview fallback still works when signing is disabled.
- Automated tests cover signed pass fields and the Wallet web-service lifecycle.
