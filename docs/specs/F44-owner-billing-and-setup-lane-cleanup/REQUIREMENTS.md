# F44 Requirements

## Functional

1. Signup billing page shows explicit owner/billing sub-step progression in a compact component.
2. Owner stage emphasizes credential setup only; billing stage emphasizes payment confirmation with owner status shown compactly.
3. Workspace onboarding mode shows one primary next-action card rather than multi-card taskboard duplication.
4. Shop setup remains editable via setup blade, including logo upload, with no route contract change.
5. Roadmap remains contract-driven and visually primary for overall step context.

## Non-Functional

1. Reduce redundant prose and duplicate progress indicators.
2. Preserve existing backend contracts and URLs.
3. Build + QA scripts remain green.
