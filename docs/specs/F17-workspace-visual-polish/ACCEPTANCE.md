# F17 Workspace Visual Polish Acceptance

## Acceptance Criteria

1. The active plan is `F17-workspace-visual-polish`, and `F16` has moved to completed plans.
2. The merchant workspace uses a tighter, more constrained content shell than the F16 version.
3. The programme rail reads as navigation and no longer repeats customer and unlock counts on each programme row.
4. The selected-programme header is compact and does not restate the entire working model.
5. `Configure` presents customer delivery as part of programme setup, with wallet loyalty card shown as the current live output.
6. Updated component tests pass.
7. The F17 validator passes.

## Demo Walkthrough

1. Open a merchant workspace and switch to `Programmes`.
2. Confirm the page feels tighter and less stretched than the F16 version.
3. Confirm the left rail behaves like a programme selector rather than an explainer panel.
4. Open `Configure` and confirm wallet loyalty card is presented as the current programme delivery option.
5. Run the F17 validator and confirm build plus tests pass.
