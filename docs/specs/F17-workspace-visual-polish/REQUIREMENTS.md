# F17 Workspace Visual Polish Requirements

## Objective

Reduce the remaining visual clutter in the merchant workspace so the programme rail reads as navigation plus one focused working surface.

## Functional Requirements

### Workspace Shell

- the merchant workspace should use a more constrained and intentional content width
- shop-level actions should stay secondary to the main workspace navigation
- the overall workspace should feel calmer without changing the existing hierarchy

### Programme Rail

- the programme rail should read as navigation, not a second content page
- programme rows should prioritise selection, lifecycle, and date window over repeated metrics
- programme creation should stay explicit about the current delivery type while remaining lightweight

### Selected Programme Work Pane

- the selected-programme header should stay compact
- operate and configure should keep the same capabilities while reducing repeated labels and metadata
- configure should present delivery as an option within the programme, with the wallet loyalty card as the current live output

## Non-Goals

- changing the merchant hierarchy again
- introducing new programme delivery types
- redesigning home, signup, or billing
