# F17 Workspace Visual Polish

## Goal

Make the merchant workspace feel more intentional and less crowded without changing the `shop -> programmes -> selected programme` hierarchy.

## Why Now

- `F16` made the work surface calmer, but the page still feels too stretched and card-heavy
- the programme rail still reads partly like content instead of navigation
- the selected-programme area still carries more framing and metadata than the merchant needs

## Scope

- tighten the merchant workspace shell and spacing
- make the programme rail read more like navigation
- reduce repeated programme counts and metadata inside the working surface
- present wallet loyalty card as the current programme delivery option without over-explaining it

## Planned Outcomes

- the merchant workspace feels more like an app shell than a dashboard
- the programme rail is easier to scan
- the right-hand work pane feels focused and less stacked
- configure keeps the wallet card present while making delivery feel like a sub-option of programme setup

## Exit Criteria

- `F17` spec exists with requirements and acceptance
- `F16` moves to completed plans
- the merchant workspace is visually calmer with the same hierarchy
- validator and tests pass
