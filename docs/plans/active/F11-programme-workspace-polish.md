# F11 Programme Workspace Polish

## Goal

Make the merchant workspace feel calmer and more operational by turning `Programmes` into a genuine selection rail, reducing stacked panel density, and moving shop settings into a lighter in-context flow.

## Why Now

- `F10` corrected the hierarchy, but the workspace still feels visually crowded in the daily-use path
- `Programmes` is now the right section, but the screen still behaves too much like content panels arranged side by side instead of navigation plus one working surface
- shop settings currently interrupt the flow more than they should for a merchant who is mainly operating programmes
- the right-side surface still over-explains the model when merchants mostly just need to run the tool

## Scope

- keep the `Shop -> Programmes -> Operate/Configure/Insights` hierarchy from `F10`
- turn the left programme column into a tighter selection rail with grouped lifecycle sections
- keep programme creation explicit about the current delivery output while reducing its visual weight
- move shop settings into a lighter in-context drawer or overlay rather than a competing full workspace page
- simplify the selected programme header into a compact context bar
- reduce redundant explanatory panels and repeated framing inside `Programmes -> Configure`
- keep the current wallet loyalty card preview, but make it feel like part of one work canvas rather than a separate explanatory surface

## Planned Outcomes

- the merchant can scan programmes quickly from a dedicated rail
- the right side reads as one work surface for the selected programme
- shop settings no longer feels like a separate page competing with loyalty operations
- the interface keeps the programme model explicit without making the page instructional
- the merchant workspace feels more like a day-to-day tool and less like a product walkthrough

## Exit Criteria

- `F11` spec exists with explicit requirements and acceptance
- `F10` has moved to completed plans
- shop settings opens as a lighter in-context surface instead of only as a dedicated workspace page
- the programmes rail is visually simpler and avoids large explanatory blocks
- the selected programme surface is calmer and avoids duplicated summary copy
- tests and validator pass after the workspace polish changes
