# F15 Programme Workspace Focus

## Goal

Make the merchant programme area feel like a daily-use tool instead of a page explaining its own model.

## Why Now

- the hierarchy is now broadly correct, but the programme workspace still feels crowded
- the left rail is still half navigation and half explanation
- the selected-programme header still spends space explaining what tabs do
- configure still competes with too many supporting panels at once

## Scope

- turn the left programmes column into a true navigation rail
- move shop settings into one clear shop-level entrypoint near the shop context
- reduce the selected-programme banner to current context only
- simplify programme configure so the form and preview are the primary focus
- keep the wallet loyalty card explicit as the current customer output, not the meaning of a programme

## Planned Outcomes

- the programmes screen feels lighter and more direct
- the left rail reads as selection and creation, not content
- the right side reads as one focused work area
- shop settings no longer compete as a floating secondary page action

## Exit Criteria

- `F15` spec exists with requirements and acceptance
- `F14` moves to completed plans
- the programme workspace reduces narrative and duplicate affordances
- validator and component tests pass
