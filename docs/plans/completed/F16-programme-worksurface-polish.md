# F16 Programme Worksurface Polish

## Goal

Reduce the visual density inside the selected-programme work surface so day-to-day operations and configuration feel faster and calmer.

## Why Now

- `F15` corrected the programme rail and header, but the right-hand work area still feels stacked and busy
- `Operate` still spreads the merchant's daily actions across too many full panels
- `Configure` still asks the merchant to parse headings, copy, form sections, and preview chrome all at once

## Scope

- simplify the `Operate` layout and supporting copy
- tighten the `Configure` layout so the form and preview read as one tool surface
- reduce non-essential labels and chips inside the selected-programme area
- keep programme insights untouched unless needed for consistency

## Planned Outcomes

- daily-use actions are easier to scan
- the selected programme surface feels less like a dashboard and more like a tool
- configure looks calmer without losing the wallet-card preview

## Exit Criteria

- `F16` spec exists with requirements and acceptance
- `F15` moves to completed plans
- the programme work surface becomes visually lighter without changing the underlying hierarchy
- validator and tests pass
