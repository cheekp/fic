# Runbooks

Use runbooks for operator-facing steps that should be repeatable without rediscovering context in code or old threads.

## Current Guides
- `APPLE_WALLET_LOCAL_DEMO.md` end-to-end local founder demo setup for adding a signed Wallet pass from an iPhone
- `NEXTJS_DESIGN_PASS_GUARDRAILS.md` design-pass scope, QA, and baseline rules for Next.js frontend refinement
- `PR_REVIEW_EVIDENCE_GUIDE.md` pull-request evidence expectations for visual hotspots, contract changes, and slice scope
- `UX_QA_PLAYBOOK.md` fast and browser-assisted UX quality gate workflow for frontend changes
